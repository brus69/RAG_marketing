[1. НАЗНАЧЕНИЕ]
Спланировать и провести минимальную валидацию гипотезы продукта до написания кода; определить метрики успеха и следующие шаги.

[2. ПРИНЦИПЫ MVP-ВАЛИДАЦИИ]
- Тестируй ценность, а не функциональность
- Измеряй поведение, а не намерения ("хотел бы" ≠ "купил")
- Запускай быстро: 7–14 дней на один эксперимент
- Учись на провалах: "не взлетело" = данные, а не неудача

[3. ТИПЫ ЭКСПЕРИМЕНТОВ (по возрастанию сложности)]

3.1 Концептуальные (0 кода)
- Лендинг с описанием ценности + CTA ("Узнать первым", "Предзаказ")
- Опрос/интервью с прототипом (Figma, скетч, текстовое описание)
- Пост в соцсетях/форуме: "Решили бы вы {проблему} за {цена}?"

3.2 Поведенческие (минимальный код)
- Fake door test: кнопка "Купить" → сбор контактов + сообщение "Скоро запуск"
- Wizard of Oz: ручное выполнение "за кулисами" при автоматическом интерфейсе
- Concierge MVP: персональное решение для 3–5 клиентов вручную

3.3 Функциональные (рабочий прототип)
- Кликабельный прототип с 1–2 ключевыми сценариями
- Урезанная версия продукта (одна фича, один сегмент)
- Пилот с 1–3 платными клиентами (дисконтированный доступ)

[4. МЕТРИКИ УСПЕХА (выбрать 1–2 ключевых)]

4.1 Для концептуальных тестов
- Конверсия в действие: % посетителей, оставивших контакт (цель: ≥5–15%)
- Качество лидов: % целевых (по критериям ЦА) среди оставивших контакт
- Обратная связь: % положительных/отрицательных реакций в комментариях

4.2 Для поведенческих тестов
- Активация: % пользователей, выполнивших ключевое действие
- Удержание: % вернувшихся через 3/7 дней
- Готовность платить: % согласившихся на предзаказ/депозит

4.3 Для функциональных тестов
- Юнит-экономика: CAC < LTV, положительная маржинальность
- Скорость роста: неделя к неделе (если применимо)
- Качественные инсайты: что удивило, что сломалось, что просят

[7. ПРОВЕРКА КАЧЕСТВА ПЛАНА]
□ Гипотеза сформулирована проверяемо (есть метрика и порог)?
□ Эксперимент соответствует стадии (не строим MVP, если не проверили ценность)?
□ Метрики поведенческие, а не опросные ("купил" ≠ "хотел бы")?
□ План включает интерпретацию и следующие шаги для обоих исходов?
□ Бюджет и сроки реалистичны для соло-фаундера/малой команды?
