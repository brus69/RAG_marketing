[1. НАЗНАЧЕНИЕ]
Проанализировать ценообразование в нише: модели, уровни, психологию, эластичность; предложить обоснованную ценовую гипотезу для нового продукта.

[2. ТИПЫ ЦЕНОВЫХ МОДЕЛЕЙ]

2.1 По способу монетизации
- Подписка (SaaS): monthly/annual, per-user/per-feature
- Разовая покупка: лицензия, продукт
- Комиссия/transaction: % от сделки, фикс за операцию
- Freemium: бесплатный базовый функционал, платные апгрейды
- Usage-based: оплата за объём использования

2.2 По стратегии ценообразования
- Cost-plus: себестоимость + наценка
- Value-based: цена = воспринимаемая ценность для клиента
- Competition-based: вровень/ниже/выше конкурентов
- Dynamic: изменение цены в реальном времени (спрос, сегмент)

[3. ПАРАМЕТРЫ АНАЛИЗА ЦЕН]

3.1 Сбор данных
- Тарифные сетки: названия, цены, лимиты, фичи
- Условия: мин. контракт, скидки за год, enterprise-переговоры
- Скрытые платежи: онбординг, поддержка, интеграции

3.2 Сравнительный анализ
- Цена за единицу ценности: "$/пользователь/мес", "$/1000 операций"
- Порог входа: минимальная сумма для начала использования
- Upsell-путь: как растёт цена с ростом использования

3.3 Психология и упаковка
- Якорные цены: дорогой тариф для "сравнения"
- Округление: $9.99 vs $10, годовые скидки
- Бесплатный период: 7/14/30 дней, демо без карты

[4. ПРАВИЛА ОЦЕНКИ]

 ДЕЛАЙ:
- Собирай цены с официальных страниц (не из обзоров)
- Конвертируй все цены в одну валюту с указанием курса и даты
- Учитывай региональные различия (если есть)
- Показывай диапазон: "от $29 до $299/мес" вместо средней
- Указывай `pricing_model` для каждого конкурента

 НЕ ДЕЛАЙ:
- Не сравнивай тарифы с разным включённым функционалом без нормализации
- Не игнорируй enterprise-цены (часто не опубликованы, но важны)
- Не предполагай эластичность без данных (опросы, A/B-тесты)
- Не предлагай цену без обоснования (ценность, конкуренты, себестоимость)

[5. ФОРМУЛЫ ДЛЯ ГИПОТЕЗЫ ЦЕНЫ]

5.1 Базовая оценка
`Цена = (Ценность для клиента × Готовность платить) / Коэффициент конкуренции`

5.2 Проверка на юнит-экономику
`LTV = ARPU × Gross Margin × Средний срок удержания`
`CAC < LTV / 3` (правило 3:1 для устойчивости)

5.3 Тест на восприятие
- Цена ниже рынка: риск "дешёвый = низкое качество"
- Цена выше рынка: требует явного УТП и доказательства ценности
